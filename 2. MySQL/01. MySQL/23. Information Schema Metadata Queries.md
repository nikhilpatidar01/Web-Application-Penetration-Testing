
## üìò `information_schema` Metadata Queries in MySQL

### üîç What is `information_schema`?

`information_schema` is a **system database** that stores **metadata** ‚Äî i.e., information **about other databases**, tables, columns, indexes, etc., managed by the MySQL server.

---

## üß† What Is a Schema?

A **schema** is just a logical container for:

* üìã Tables
* üëÄ Views
* üóÉ Indexes
* ‚öô Stored Procedures
* üß® Triggers
* üî£ Functions
* üïì Events
* üõ° Privileges

### üí° Note:

In **MySQL**, **`SCHEMA` and `DATABASE` mean the same thing**.

```sql
CREATE DATABASE nikhil_db;
-- is the same as
CREATE SCHEMA nikhil_db;
```

---

## üìÇ Access & Explore Metadata with Queries

### üî∏ View All Schemas (Databases)

```sql
SHOW DATABASES;
```

---

### üî∏ Use a Schema

```sql
USE nikhil_db;
```

---

### üî∏ List All Tables in a Schema

```sql
SHOW TABLES;
```

Or using `information_schema`:

```sql
SELECT TABLE_NAME
FROM information_schema.TABLES
WHERE TABLE_SCHEMA = 'nikhil_db';
```

---

### üî∏ See Schema Details

```sql
SELECT * FROM information_schema.SCHEMATA;
```

üîç Shows all schemas with collation, charset, etc.

---

### üî∏ List All Columns in a Table

```sql
SELECT COLUMN_NAME
FROM information_schema.COLUMNS
WHERE TABLE_NAME = 'users'
  AND TABLE_SCHEMA = 'nikhil_db';
```

---

## üß™ Explore Metadata via `information_schema`

### üßæ Show All Tables in `information_schema`

```sql
USE information_schema;
SHOW TABLES;
```

üëÄ Includes: `TABLES`, `COLUMNS`, `SCHEMATA`, `STATISTICS`, `KEY_COLUMN_USAGE`, etc.

---

### üîé Table Metadata Examples

#### All Table Info

```sql
SELECT * FROM TABLES;
```

#### Schema + Table Names

```sql
SELECT TABLE_SCHEMA, TABLE_NAME FROM TABLES;
```

#### All Schemas (Grouped)

```sql
SELECT TABLE_SCHEMA FROM TABLES GROUP BY TABLE_SCHEMA;
```

#### Filter Tables by Schema

```sql
SELECT TABLE_NAME FROM TABLES WHERE TABLE_SCHEMA = 'nikhil_db';
```

```sql
SELECT TABLE_NAME FROM TABLES WHERE TABLE_SCHEMA = 'mysql';
```

---

### üîé Column Metadata Examples

#### All Column Info

```sql
SELECT * FROM COLUMNS;
```

#### List Schemas in Columns Table

```sql
SELECT TABLE_SCHEMA FROM COLUMNS GROUP BY TABLE_SCHEMA;
```

#### List Tables from a Schema

```sql
SELECT TABLE_NAME FROM COLUMNS
WHERE TABLE_SCHEMA = 'nikhil_db'
GROUP BY TABLE_NAME;
```

#### List Columns of a Specific Table

```sql
SELECT COLUMN_NAME FROM COLUMNS
WHERE TABLE_NAME = 'users'
  AND TABLE_SCHEMA = 'nikhil_db';
```

Or simplified:

```sql
SELECT COLUMN_NAME FROM COLUMNS WHERE TABLE_NAME = 'users';
```

---

## üßæ Redundant Yet Valid (Alternative) Queries

All of the following are still valid:

```sql
-- Redundant Yet Valid Queries: These return the same metadata using fully qualified table names

SELECT TABLE_SCHEMA FROM information_schema.TABLES GROUP BY TABLE_SCHEMA;

SELECT TABLE_NAME FROM information_schema.TABLES WHERE TABLE_SCHEMA = 'nikhil_db';

SELECT COLUMN_NAME FROM information_schema.COLUMNS WHERE TABLE_NAME = 'users';

SELECT COLUMN_NAME FROM information_schema.COLUMNS WHERE TABLE_NAME = 'users' AND TABLE_SCHEMA = 'nikhil_db';

```
